## A1-KI-001：Live 模式耗时/token 基线未封

**现象**

Live backend 受模型服务、网络、提示词长度、玩家数和阶段并发影响。目前还需要固定 5 人 live 短局的耗时与 token/调用量基线。

**影响范围**

- 首夜、白天讨论、提名、顺序投票、夜晚行动可能出现等待时间波动。
- 内测组织者暂时不能给出稳定的单局时长和模型成本预估。
- 若模型返回空响应、非法结构或超时，主链依赖 fallback 质量。

**发布判断**

- 不建议作为“稳定时长”承诺对外发布。
- 发布前至少需要一次 5 人 live 短局完成首日处决，并记录阶段耗时。
- 若 token/调用量样本仍未封，应在 release checklist 中标注未完成。

**临时规避**

- 内测演示优先使用 mock 模式或低人数 live 短局。
- Live 局提前说明模型响应时间可能波动。
- 每次 live smoke 记录 `game_id`、模型配置、阶段耗时和异常 fallback。

**后续去向**

- 对应 `A1-LIVE-005` 和 `A1-PERF-011`。
- 在 M3/M6 中固定耗时统计、调用量记录和诊断入口。

## A1-KI-002：浏览器级真人验收记录未沉淀

**现象**

仓库已有 repo-local 前端验收入口，但浏览器级真人/半真人 smoke 需要人工记录，包括玩家端和说书人端的实际操作流程。

**影响范围**

- 真人提名、辩解、投票、聊天、状态页、私密信息、结算历史等连续体验缺少发布候选记录。
- 玩家视角隔离和说书人魔典访问需要浏览器级确认。
- 窄屏和不同浏览器的可用性风险尚未完全量化。

**发布判断**

- 若完全没有 5 人真人/半真人浏览器验收记录，应视为阻断 `alpha1.0` 内测标签。
- 若已有一次通过记录但覆盖不足，可带出为 P1 known issue，并限制首轮内测范围。

**临时规避**

- 发布前按 `docs/alpha-1.0-release-checklist.md` 补一份 5 人真人/半真人记录。
- 将浏览器、窗口尺寸、`game_id`、玩家模式、说书人模式和异常截图路径写入 checklist。

**后续去向**

- 对应 `A1-FRONT-002`、`A1-SEC-006` 和 `A1-UX-010`。
- 将可重复的浏览器流程逐步固化为 Playwright 或 MCP smoke。

## A1-KI-003：数据目录与日志体积控制待整理

**现象**

项目会产生对局历史、AI traces、说书人裁量、快照、测试运行数据和 session 日志。`docs/alpha-1.0-data-operations.md` 已整理哪些目录可清理、哪些资产必须随 `game_id` 保留，但内测执行时仍需要按说明操作。

**影响范围**

- 多局内测后 `data/`、session 日志和导出资产可能快速增长。
- 问题定位时可能需要人工筛选多个记录源。
- 发布包或工作区可能夹带临时测试数据。

**发布判断**

- 不阻断小范围内测，但必须在发布 checklist 中确认清理策略。
- 若无法通过 `game_id` 定位关键资产，则应阻断发布。

**临时规避**

- 内测前记录干净数据目录基线。
- 每局保留 `game_id`，按需运行导出脚本。
- 不把临时测试数据作为发布资产提交。

**后续去向**

- 对应 `A1-DATA-008` 和 `A1-PERF-011`。
- 已在 M6 中补充数据目录说明、可清理列表和问题包导出流程。

## A1-KI-004：发布口径仍在更新中

**现象**

README、alpha1.0 计划、阶段任务板、release checklist、known issues 和 changelog/version notes 需要保持一致。当前 README、CHANGELOG、VERSION_NOTES、release checklist、known issues 已转向 alpha1.0 内测候选口径；打标签前仍需最后复核。

**影响范围**

- 内测参与者可能看到 alpha0.3 或 alpha1.0 混合描述。
- 验收入口可能因脚本仍在并行开发而出现“计划存在、命令未落地”的短期差异。
- 对外摘要需要避免过度承诺。

**发布判断**

- 不阻断文档继续收束。
- 打 `alpha1.0` 标签前必须确认 README、checklist、known issues、CHANGELOG 或版本说明口径一致。

**临时规避**

- 对外以 `docs/alpha-1.0-release-checklist.md` 和 README 为准。
- 尚未落地的命令只写为候选或 N/A，不写成已通过事实。
- 明确 Alpha 1.0 是内测候选，不承诺生产级稳定。

**后续去向**

- 对应 `A1-QA-009` 和 M6 发布工程。
- 发布冻结前复核 changelog/version notes 与 checklist 记录。
