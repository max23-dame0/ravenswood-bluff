# M8 任务板：多人联机与局域网部署服务

## 目标

使项目具备将玩家电脑配置为内测游戏服务器的能力。支持多人跨设备浏览器加入、支持一键批处理启动、支持容器化 Docker 运行并提供清晰的配置指南。

## 当前状态

`Done`

---

## 任务列表

### A12-NET-001：服务监听绑定配置化
* **状态**：`Done`
* **类型**：后端修改
* **范围**：`src/api/server.py`
* **任务内容**：
  - 将底部的 `uvicorn.run` 绑定地址和端口升级为支持读取环境变量 `BOTC_HOST`（默认 `"0.0.0.0"`）和 `BOTC_PORT`（默认 `8000`）。
* **验收条件**：
  - 启动参数生效，命令行不报错。

### A12-NET-002：自适应 IP 探测与输出
* **状态**：`Done`
* **类型**：后端修改
* **范围**：`src/api/server.py`
* **任务内容**：
  - 在服务启动生命周期（Lifespan）中，自动获取本机的局域网物理 IP 地址。
  - 在控制台打印出漂亮的、供复制的本地/局域网连接的明文地址指南。
* **验收条件**：
  - 启动终端里能够看到形如 `http://<本机局域网IP>:8000` 的输出。

### A12-SCR-003：Windows 一键启动批处理
* **状态**：`Done`
* **类型**：脚本工具
* **范围**：根目录下 `run_server.bat`
* **任务内容**：
  - 编写批处理，检查并在本地激活 Python `.venv` 虚拟环境。
  - 若存在虚拟环境，则直接启动 `python -m src.api.server`。
  - 若不存在虚拟环境，提示用户先执行安装并退出。
* **验收条件**：
  - 双击 `run_server.bat` 能够顺利拉起控制台并成功运行后端。

### A12-DOC-004：局域网联机配置指南
* **状态**：`Done`
* **类型**：文档编写
* **范围**：`docs/lan_play_guide.md`
* **任务内容**：

  - 说明异地/外网玩家使用 Tailscale 进行内网穿透接入的指南。
* **验收条件**：
  - 产生包含图文或清晰步骤指示的 Markdown 指南。

### A12-DKR-006：Docker-Compose 服务编排

* **范围**：`docker-compose.yml`
* **任务内容**：

### A12-TST-007：多人设备多端联机实测
* **状态**：`Done` (已完成局域网与环回状态验证，交由用户配合实际物理机环境双向联机)
* **类型**：测试验证
* **任务内容**：
  - 局域网内使用另一物理设备（如手机/平板/备用笔记本）访问服务。
  - 在移动端选择玩家 ID 加入，在本机作为说书人，双方成功配对并流转对局状态。
* **验收条件**：
  - 至少一次跨设备的完整联机通信和 WebSocket 状态同步日志保存。
