# Alpha 1.2 开发计划：Network Hosting & Multiplayer Readiness

## 1. 版本定位

Alpha 1.1 解决了“AI 玩家难度选择与响应流畅体验”。Alpha 1.2 要解决的是 **“多人设备联机与云部署准备”**。

在 Alpha 1.2 中，我们将通过把主机的电脑作为服务器（利用主机上的 API 密钥及环境配置），让其他内测玩家通过各自的浏览器（电脑/平板/手机）无门槛地加入同一个多人对局中。

**一句话目标：**
**打通设备壁垒，支持玩家使用本机作为服务器进行多人/局域网联机，并提供极简一键运行脚本与 Docker 支持，为后续真正的公网/云端发布扫清技术障碍。**

---

## 2. 现状分析

### 2.1 目前做得好的地方
- **前端网络解耦**：前端的 [index.html](file:///d:/鸦木布拉夫小镇/public/index.html) 与 [storyteller.html](file:///d:/鸦木布拉夫小镇/public/storyteller.html) 在通信起源上均采用了动态 Origin 获取方法 (`window.location.hostname`、`window.location.port`)，天然支持跨 IP 连接。
- **多人/多连接接入**：FastAPI 后端和 WebSocket 管理器已具备支持多个不同 `player_id` 建立连接的能力，为多人混合局提供了底层通信框架。

### 2.2 需要改进的痛点

| 问题 | 根因 | 影响 |
|------|------|------|
| **接口硬编码限制** | [server.py](file:///d:/鸦木布拉夫小镇/src/api/server.py) 主入口 `uvicorn.run` 硬编码绑定为 `127.0.0.1` | 其他设备（局域网/公网）无法通过主机的物理 IP 访问 Web 服务，只能主机自己玩。 |
| **接入链接不透明** | 启动日志仅打印 `127.0.0.1`，普通用户不会查找局域网 IP | 主机不知道该把哪个 URL 链接发给朋友，导致联机配置成本极高。 |
| **启动门槛高** | 每次都需要手动通过命令行进入虚拟环境、启动 python 模块 | 非技术内测人员想当 Host 时无从下手，无法一键拉起服务。 |
| **缺少容器化支撑** | 项目中缺少 Docker 配置 | 不利于在其他服务器环境进行一键化快速部署。 |
| **防火墙与安全指引空缺** | 缺少操作系统级的端口开放和联机配置指导 | 玩家经常会因为 Windows 防火墙阻断连接而联机失败，找不到排查指南。 |

## 3. 设计方案

### 3.1 服务网络绑定与自适应 IP 探测
我们不修改对局逻辑和已有的接口契约，只在网络服务层做扩展：
1. **环境变量控制端口与绑定**：
   - 将 `uvicorn.run` 中的 `host` 改为读取 `BOTC_HOST`（默认 `"0.0.0.0"`，使其监听所有网卡接口）。
   - 将 `port` 改为读取 `BOTC_PORT`（默认 `8000`）。
2. **局域网物理 IP 自检测与输出**：
   - 在 FastAPI lifespan 启动输出中，使用 `socket` 模块获取本机的局域网物理 IP 地址（过滤掉回路和虚拟网卡）。
   - 动态打印出玩家可复制的加入路径：
     ```text
     ========================================================================
     鸦木布拉夫小镇 (Ravenswood Bluff) 多人联机服务启动成功！
     
       - 本地直接访问： http://127.0.0.1:8000
       - 局域网玩家访问： http://192.168.1.100:8000  (请将其复制发送给您的内测朋友)
       - 说书人控制台：   http://127.0.0.1:8000/ui/storyteller.html
     ========================================================================
     ```

### 3.2 一键式启动器 (`run_server.bat`)
在项目根目录下创建一个易于运行的 Windows 批处理脚本：
- 自动检查当前目录下 `.venv` 的存在性，激活虚拟环境。
- 提供简单友好的错误输出提示（如未配置虚拟环境，提示先运行 `pip install`）。
- 启动 `src.api.server` 模块，将日志同步打印在命令行中。

### 3.3 Docker 容器化支持
编写容器化配置以备后续真正上线云端：
- **`Dockerfile`**：采用轻量级 `python:3.11-slim` 基础镜像，暴露 `8000` 端口。
- **`docker-compose.yml`**：对外部环境变量 `.env` 进行直通，挂载运行时产生的日志（`storyteller_run.log`、`orchestrator_run.log`）及数据库。

### 3.4 局域网联机指引文档
在 `docs/` 下编写一份面向玩家主机的 `lan_play_guide.md`，主要涵盖：
- Windows 系统如何找到自己电脑的局域网 IP。
- 如何设置 Windows 防火墙出入站规则（开放 8000 端口，或允许 python 访问）。
- 如何使用 VPN/内网穿透（如 Tailscale）进行跨互联网公网联机。

## 7. 验收标准

### 启动命令与输出验证
双击 [run_server.bat](file:///d:/鸦木布拉夫小镇/run_server.bat) 或在命令行执行：
```powershell
.\.venv\Scripts\python.exe -m src.api.server
```
控制台必须正确打印出：
1. 本地回路访问链接。
2. 以当前物理局域网网卡 IP 为前缀的外部接入链接（如 `http://192.168.x.x:8000`）。
3. 且外部链接确为有效物理局域网 IP（不是 `127.0.0.1` 或是虚拟网卡 IP）。

### 网络通信验证
1. 本机浏览器访问说书人控制台，其他局域网物理设备打开浏览器访问玩家加入链接。
2. 局域网设备成功建立 WebSocket 握手，断开重连正常。
3. 对局交互日志能实时被所有已连接设备同步。

## 8. 对外发布摘要草案

Alpha 1.2 专注于打通本地设备限制，让玩家电脑变成内测服务器。新版本通过自动探测物理局域网 IP、开放全局网络监听绑定，消除了其他测试玩家的 Python 环境门槛。

核心改进：
- **0.0.0.0 默认绑定**：服务不再限制在 127.0.0.1，局域网同 WiFi 玩家直接可连。
- **自动 IP 提示**：启动时终端明文提示测试链接，降低了寻找 IP 的复杂度。
- **一键批处理**：新增 Windows 平台下一键启动器，双击即可进入对局。
- **Docker 容器化**：补齐 Docker 与 Docker-Compose 配置文件，支持跨平台一键部署。
- **联机白皮书**：随版本附赠防火墙开放与异地联机（Tailscale）配置指南，为非技术内测保驾护航。
